in package

Выводит список новостей.

Компонент выводит список анонсов новостей.

Если указан ID или символьный код инфоблока, будут выведены новости этого инфоблока. В противном случае будут выведены новости всех инфоблоков указанного типа.

Параметры компонента:

  • IBLOCK_TYPE – ID типа инфоблока. Если заданы IBLOCK_ID или IBLOCK_CODE, данный параметр игнорируется.
  • IBLOCK_ID – ID инфоблока.
  • IBLOCK_CODE – Символьный код инфоблока. Если задан IBLOCK_ID, данный параметр игнорируется.
  • SECTION_ID – ID раздела, новости которого надо вывести. Если SECTION_ID и SECTION_CODE не заданы, выводятся новости корневого раздела инфоблока.
  • SECTION_CODE – Символьный код раздела, новости которого надо вывести. Если задан SECTION_ID, данный параметр игнорируется. Если SECTION_ID и SECTION_CODE не заданы, выводятся новости корневого раздела инфоблока.
  • INCLUDE_SUBSECTIONS – Если значение параметра равно "Y", то выводятся новости подразделов. Если значение равно "N", то новости подразделов не выводятся. По умолчанию "Y".
  • DETAIL_URL – Шаблон адреса страницы детального просмотра новости. По умолчанию значение берётся из инфоблока.
  • SECTION_URL – Шаблон адреса страницы раздела новостей. По умолчанию значение берётся из инфоблока.
  • IBLOCK_URL – Шаблон адреса страницы инфоблока. По умолчанию значение берётся из инфоблока.
  • FILTER – Ассоциативный массив, по которому будут отфильтрованы новости перед выводом на страницу. Если параметр не задан, фильтрация не производится.
  • FILTER_CACHE – Если параметр равен "Y", то при использовании фильтра компонент будет кэшироваться. Если параметр равен "N", то при использовании фильтра компонент кэшироваться не будет. По умолчанию "N".
  • CACHE_TYPE – Тип кэширования компонента. Допустимые значения: "A" – автоматическое кэширование, "Y" – управляемое кэширование, "N" – без кэширования. По умолчанию "A".
  • CACHE_TIME – Время кэширования в секундах. По умолчанию 36000000.

Для корректной работы компонента необходимо задать хотя бы один из параметров IBLOCK_TYPE, IBLOCK_ID, IBLOCK_CODE.
